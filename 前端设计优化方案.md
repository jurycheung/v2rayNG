## 🎯 设计理念

### 核心概念
v2rayNG 不是普通应用，它是**数字时代的网络工具**——专业、精确、可靠。设计应该反映这种技术本质，同时避免generic AI 美学。

### 美学方向
**Cyber-Industrial Utilitarian** - 融合：
- 🏭 **工业工具美学** - 功能至上，精确，可靠
- 🔮 **赛博朋克元素** - 科技感，未来感，网络空间
- ⚡ **极简主义** - 去除冗余，专注核心功能

### 差异化要点
**让人记住的是什么？**
> "这不是一个漂亮的代理应用，这是一个**精密的网络仪器**"

```kotlin
// MainActivity.kt - 连接时脉冲动画
private fun startPulseAnimation() {
    val pulseAnim = ValueAnimator.ofFloat(1f, 1.2f, 1f)
    pulseAnim.duration = 2000
    pulseAnim.repeatCount = ValueAnimator.INFINITE
    pulseAnim.addUpdateListener { animator ->
        val scale = animator.animatedValue as Float
        binding.fab.scaleX = scale
        binding.fab.scaleY = scale
        binding.fab.alpha = 0.7f + (scale - 1f) * 0.3f
    }
    pulseAnim.start()
}
```

## 🎬 动画效果

### 1. 页面加载 - 级联淡入

```kotlin
// BaseActivity.kt
protected fun animateViewHierarchy(root: ViewGroup, delay: Long = 0) {
    for (i in 0 until root.childCount) {
        val child = root.getChildAt(i)
        child.alpha = 0f
        child.translationY = 20f
        child.animate()
            .alpha(1f)
            .translationY(0f)
            .setStartDelay(delay + i * 50)
            .setDuration(300)
            .setInterpolator(DecelerateInterpolator())
            .start()
        
        if (child is ViewGroup) {
            animateViewHierarchy(child, delay + i * 50)
        }
    }
}
```

### 2. 服务器切换 - 平滑过渡

```kotlin
// MainRecyclerAdapter.kt
private fun animateServerSelection(holder: ServerViewHolder, isSelected: Boolean) {
    if (isSelected) {
        // 选中 - 边框高亮动画
        val borderAnim = ValueAnimator.ofFloat(0f, 1f)
        borderAnim.duration = 300
        borderAnim.addUpdateListener { animator ->
            val progress = animator.animatedValue as Float
            holder.itemView.alpha = 0.5f + progress * 0.5f
            holder.itemView.scaleX = 1f - progress * 0.02f
            holder.itemView.scaleY = 1f - progress * 0.02f
        }
        borderAnim.start()
    }
}
```

### 3. 连接/断开 - 粒子爆发效果

```kotlin
// 使用 Lottie 或自定义 View 实现
// 简化版本：使用多个 View 模拟粒子
private fun showConnectionParticles() {
    for (i in 0 until 12) {
        val particle = View(this).apply {
            layoutParams = ViewGroup.LayoutParams(8, 8)
            background = ContextCompat.getDrawable(this@MainActivity, R.drawable.particle)
        }
        binding.root.addView(particle)
        
        // 随机方向飞散
        val angle = (i * 30).toDouble().toRadians()
        val distance = 200
        
        particle.animate()
            .translationX(cos(angle) * distance.toFloat())
            .translationY(sin(angle) * distance.toFloat())
            .alpha(0f)
            .setDuration(600)
            .setInterpolator(AccelerateDecelerateInterpolator())
            .withEndAction { binding.root.removeView(particle) }
            .start()
    }
}
```

## 📊 设计原则总结

```
✅ DO:
- 使用深色主题，增加专业感
- 强调色使用霓虹色（青色/绿色）
- 添加微妙的动画反馈
- 使用单色字体显示数据
- 创造视觉层次和深度

❌ DON'T:
- 使用纯黑 (#000000)
- 使用通用 Material Design 配色
- 过度使用阴影
- 使用系统默认字体
- 忽略动画和过渡效果
```

---

**设计愿景：**
> v2rayNG 应该像一个精密的网络仪器，而不是一个普通的代理应用。
> 每一次连接都应该让人感到可靠和专业，每一个动画都应该有意义和目的。
